# Phase 1.1: Tab 模型修正 + 檢視切換設計

**日期**: 2026-03-20
**狀態**: Draft
**前置**: Phase 1 已完成（v1.0.0-alpha.1）
**範圍**: Tab 核心模型改為開放式 + Tab Renderer Registry + 檢視切換 UI

---

## 1. 設計目標

修正 Phase 1 的 Tab 模型缺陷：`type: 'terminal' | 'stream'` 將 daemon 模式與 UI 檢視綁死，導致 session tab 無法自由切換 terminal/stream 檢視。

同時引入 Tab Renderer Registry，為未來 plugin 擴充 tab 類型奠定基礎。

### 核心原則

- **檢視切換是純 UI 操作**，不觸發 handoff API
- **Handoff 是獨立的 daemon 操作**，留在 ConversationView 內
- **Tab type 開放式**，plugin 可註冊新類型
- **不做 keep-alive**，viewMode 切換即 mount/unmount

---

## 2. Tab 模型

### 2.1 Tab Interface（開放式）

```typescript
interface Tab {
  id: string
  type: string                     // 開放字串，非 union
  label: string
  icon: string
  hostId: string
  viewMode?: string                // 支持多 view 的 tab（如 session）
  data: Record<string, unknown>    // type-specific 資料
}
```

### 2.2 內建 tab 類型的 data 結構

**session tab**:
```typescript
{
  type: 'session',
  viewMode: 'terminal' | 'stream',  // 當前檢視
  data: { sessionName: string }
}
```

**editor tab**:
```typescript
{
  type: 'editor',
  data: { filePath: string, isDirty: boolean }
}
```

### 2.3 與 v0/Phase 1 的差異

| 面向 | Phase 1 | Phase 1.1 |
|------|---------|-----------|
| Tab type | `'terminal' \| 'stream' \| 'editor'`（封閉 union） | `string`（開放） |
| Session 表示 | 兩個不同 type | 統一為 `'session'` + `viewMode` |
| Type-specific 欄位 | `sessionName?`, `filePath?`, `isDirty?` 混在 Tab 上 | 收進 `data` bag |
| 渲染方式 | TabContent 內 switch/if | Tab Renderer Registry 查詢 |
| 檢視切換 | 無（type 固定） | `viewMode` 純 UI 切換 |

## 3. Tab Renderer Registry

### 3.1 Registry 結構

```typescript
interface TabRendererConfig {
  component: React.ComponentType<TabRendererProps>
  viewModes?: string[]          // 支持的 viewMode 列表
  defaultViewMode?: string      // 預設 viewMode
  icon: (tab: Tab) => string    // 動態 icon（可依 viewMode 變化）
}

interface TabRendererProps {
  tab: Tab
  isActive: boolean
  wsBase: string
  daemonBase: string
}
```

### 3.2 內建註冊

```typescript
registerTabRenderer('session', {
  component: SessionTabContent,
  viewModes: ['terminal', 'stream'],
  defaultViewMode: 'terminal',
  icon: (tab) => tab.viewMode === 'stream' ? 'ChatCircleDots' : 'Terminal',
})

registerTabRenderer('editor', {
  component: EditorTabContent,
  icon: () => 'File',
})
```

### 3.3 TabContent 渲染邏輯

```typescript
// 從 registry 查詢，取代 switch/if
const config = tabRegistry.get(activeTab.type)
if (config) {
  return <config.component tab={activeTab} isActive={true} wsBase={...} daemonBase={...} />
}
// fallback: 未知類型
return <div>Unknown tab type: {activeTab.type}</div>
```

### 3.4 Icon 解析

Registry 的 `icon(tab)` 回傳值必須是 TabBar `ICON_MAP` 中已註冊的 key。Plugin 新增 icon 時需同步擴充 `ICON_MAP`。未來可改為動態 icon resolver，但 Phase 1.1 維持現有 Map 機制。

### 3.5 未來 plugin 擴充

Plugin 只需呼叫 `registerTabRenderer` 並擴充 `ICON_MAP` 即可新增 tab 類型：

```typescript
// 假設未來的 monitoring plugin
ICON_MAP['ChartLine'] = ChartLine  // 先註冊 icon
registerTabRenderer('monitoring', {
  component: MonitoringDashboard,
  icon: () => 'ChartLine',
})
```

## 4. SessionTabContent 元件

session 類型的 tab renderer，根據 `viewMode` 渲染 terminal 或 stream 畫面。

```typescript
function SessionTabContent({ tab, isActive, wsBase, daemonBase }: TabRendererProps) {
  const sessionName = tab.data.sessionName as string
  const viewMode = tab.viewMode ?? 'terminal'

  if (viewMode === 'terminal') {
    return <TerminalView wsUrl={`${wsBase}/ws/terminal/${sessionName}`} visible={isActive} />
  }

  return <ConversationView sessionName={sessionName} />
}
```

**不做 keep-alive**：viewMode 切換時 unmount 舊 view、mount 新 view。
- Terminal：xterm.js 銷毀重建 + WS 重連，scrollback 在 tmux 端不遺失
- Stream：messages 在 store，remount 無成本

**Handoff 按鈕**：留在 ConversationView 內（比照 v0），本 phase 不動。

## 5. StatusBar 檢視切換

### 5.1 UI 設計

StatusBar 右側顯示當前 viewMode badge，點擊展開上拉選單切換檢視。

```
│ mlab │ dev-server │ connected │                    [term ▴] │
                                                  ┌──────────┐
                                                  │ terminal ✓│
                                                  │ stream    │
                                                  └──────────┘
```

### 5.2 行為

- Badge 樣式比照 TabBar mode badge（綠色 term / 藍色 stream）
- 點擊 badge → 上拉選單，列出 registry 中該 tab type 的 `viewModes`
- 選擇後更新 `tab.viewMode`（純 UI，不觸發 handoff）
- 只有 registry config 的 `viewModes` 陣列非空時才顯示 badge（editor tab 的 registry 無 viewModes，不顯示）
- 上拉選單點擊外部關閉

### 5.3 StatusBar 其他欄位不變

- hostName、sessionName、status 維持現有邏輯
- mode 文字從 `tab.type` 改為 `tab.viewMode`

## 6. Data 存取 Helper

`sessionName`、`isDirty` 等欄位移入 `data` bag 後，需提供 typed helper 避免到處 cast：

```typescript
// lib/tab-helpers.ts
export function getSessionName(tab: Tab): string | undefined {
  return tab.data.sessionName as string | undefined
}

export function getFilePath(tab: Tab): string | undefined {
  return tab.data.filePath as string | undefined
}

export function isDirty(tab: Tab): boolean {
  return (tab.data.isDirty as boolean) ?? false
}
```

所有原本直接存取 `tab.sessionName`、`tab.isDirty` 的程式碼改用這些 helper。

## 7. Store 改動

### 7.1 useTabStore

- `createTab` helper 調整：接受 `data` 參數
- 新增 `setViewMode(tabId: string, viewMode: string)` action
- **`dismissTab`**：從 `tab.sessionName` 改為 `getSessionName(tab)` 讀取 sessionName 加入 `dismissedSessions`

### 7.2 useSessionTabSync

- 建立 tab 時改為 `type: 'session'`，`viewMode: 'terminal'`（或依 session.mode 判斷），`data: { sessionName }`

### 7.3 App.tsx 完整改動清單

- 移除 handoff 後的 `updateTab(tab.id, { type: 'terminal', icon: 'Terminal' })`
- Handoff 改變 daemon 模式，UI 檢視不跟著變（使用者自行切換 viewMode）
- `activeTab?.sessionName` → `getSessionName(activeTab)`（第 69、190 行等）
- `t.sessionName === ...` → `getSessionName(t) === ...`（第 109、152 行等）
- `t.sessionName` 傳給 SessionPicker → `getSessionName(t)`（第 240 行）
- `statusMode = activeTab?.type` → `statusMode = activeTab?.viewMode`（第 191 行）

## 8. TabBar 改動

### 8.1 Icon 動態化

目前 TabBar 直接讀 `tab.icon`。改為：

1. 優先從 registry 的 `icon(tab)` 函式取得（依 viewMode 動態變化）
2. Fallback 到 `tab.icon`

這樣 session tab 在 terminal viewMode 時顯示 Terminal icon，切到 stream 時自動變 ChatCircleDots icon。

### 8.2 isDirty 顯示

目前 TabBar 直接讀 `tab.isDirty`。改為使用 `isDirty(tab)` helper 從 `data` bag 讀取。

## 9. localStorage 遷移

現有 `tbox-tabs` 中存的 tab 格式需一次性遷移：

```typescript
// 遷移邏輯（在 useTabStore persist 的 version + migrate 中）
function migrateTab(tab: any): Tab {
  if (tab.type === 'terminal' || tab.type === 'stream') {
    const { sessionName, ...rest } = tab
    return {
      ...rest,
      type: 'session',
      viewMode: tab.type === 'stream' ? 'stream' : 'terminal',
      data: { sessionName },
    }
  }
  if (tab.type === 'editor') {
    const { filePath, isDirty, ...rest } = tab
    return {
      ...rest,
      type: 'editor',
      data: { filePath, isDirty: isDirty ?? false },
    }
  }
  return { ...tab, data: tab.data ?? {} }
}
```

遷移時 destructure 移除舊的頂層欄位（`sessionName`、`filePath`、`isDirty`），避免殘留。

Zustand persist 的 `version` + `migrate` 機制處理。
