## 1. Tab 模型擴充

### Tab Interface 變更

```typescript
interface Tab {
  id: string
  type: string
  label: string
  icon: string
  hostId: string
  viewMode?: string
  data: Record<string, unknown>
  pinned?: boolean   // 新增：固定分頁（縮小+釘左側+隱含鎖定）
  locked?: boolean   // 新增：鎖定分頁（擋關閉，不影響位置）
}
```

### 規則

- `pinned: true` 隱含 `locked: true`（固定的一定是鎖定的）
- 取消固定時不自動取消鎖定（使用者可能想保持鎖定）
- `tabOrder` 排列：固定區在最左 → 一般區在右，各區內維持使用者排序

### Store 新增方法

```typescript
// useTabStore 新增
pinTab(tabId: string): void      // 設 pinned=true, locked=true, 移到固定區末尾
unpinTab(tabId: string): void    // 設 pinned=false, 移到一般區開頭（locked 不變）
lockTab(tabId: string): void     // 設 locked=true
unlockTab(tabId: string): void   // 設 locked=false（pinned tab 不可 unlock）

// removeTab / dismissTab 修改
removeTab(tabId: string): void   // locked tab 拒絕關閉（直接 return）
dismissTab(tabId: string): void  // 同上
```

### UISettingsStore 新增

```typescript
// useUISettingsStore 新增
keepAliveCount: number              // 預設 0（切走即銷毀）
keepAlivePinned: boolean            // 預設 false（pinned tab 不特殊保活）
terminalSettingsVersion: number     // 預設 0（遞增計數器，觸發 terminal 重建）
bumpTerminalSettingsVersion(): void // 遞增 terminalSettingsVersion
```

### Persist Migration

- **useTabStore**：v1 → v2，舊 tab 補 `pinned: false, locked: false`
- **useUISettingsStore**：不需要 migration。新欄位（`keepAliveCount`, `keepAlivePinned`）有預設值，Zustand persist 的 merge 行為會自動補上

## 2. Keep-Alive 機制 + TerminalView 拆分

### Keep-Alive 架構

TabContent 從「只掛載 activeTab」改為 LRU cache pool 策略。

**Pool 管理邏輯：**

1. 切換到 tab → 將該 tab 移到 pool 最前面
2. pool 長度 > keepAliveCount → 淘汰末尾（unmount）
3. `keepAlivePinned: true` 時，pinned tab 不計入 pool 額度（永遠保活）
4. `keepAliveCount = 0` → pool 只有 activeTab（現有行為，預設）
5. Settings 變更（renderer / sizing / keepAliveCount） → 清空整個 pool，全部銷毀重建

**渲染方式：**

- Active tab：`display: block`
- Pool 中其他 tab：`display: none`（保留 DOM + WS 連線）
- 不在 pool 中：unmount

### TerminalView 拆分

現為 221 行，拆分為：

| 檔案 | 職責 | 預估行數 |
|------|------|----------|
| `hooks/useTerminal.ts` | xterm instance 初始化、addon 管理（fit/webgl）、IME 處理 | ~60 行 |
| `hooks/useTerminalWs.ts` | WS 連線建立、data relay、reconnect 偵測 | ~50 行 |
| `components/TerminalView.tsx` | 組合 hooks、overlay UI、container ref | ~100 行 |

### onTerminalReconnect 實作

**邏輯擁有者：** TabContent 層級的 pool manager（不再是 App.tsx 的 prop callback）。

**接線變更：**
- 移除 App.tsx 中的 `onTerminalReconnect={() => {}}` 空函式
- `useUISettingsStore` 新增 `terminalSettingsVersion: number`（預設 0）遞增計數器
- SettingsPanel 在任何 terminal 相關設定儲存後（包含 `terminalRenderer`、daemon config 的 `sizing_mode`、`keepAliveCount`），呼叫 `bumpTerminalSettingsVersion()`
- TabContent 只需 subscribe `terminalSettingsVersion` 一個值

**為何用計數器：** `terminalRenderer` 和 `keepAliveCount` 在 UISettingsStore，但 `sizing_mode` 在 daemon config（透過 API 儲存在 useConfigStore）。分散監聽多個 store 複雜且脆弱，統一用 version bump 更簡潔。

**觸發流程：**
1. SettingsPanel 儲存設定 → 呼叫 `bumpTerminalSettingsVersion()`
2. TabContent 透過 Zustand subscribe 偵測 `terminalSettingsVersion` 變更
3. 清空 alive pool → 所有 cached terminal 銷毀
4. Active tab 立即重建 + overlay fade 過渡

## 3. 拖曳排序

### 依賴

新增 `@dnd-kit/core` + `@dnd-kit/sortable`。

### TabBar 區域劃分

```
[固定區: icon-only tabs] | [一般區: icon + label + close] [+]
```

- 固定 tab：36px 寬，只顯示 icon
- 一般 tab：icon + label + close button
- 中間有視覺分隔線

### 拖曳規則

- 固定區內可互排、一般區內可互排
- **不能拖曳跨區**（固定/取消固定靠右鍵選單）
- 固定時自動移到固定區末尾
- 取消固定時移到一般區開頭

### 實作結構

兩個獨立 `SortableContext` 天然隔離兩區：

```tsx
<DndContext onDragEnd={handleDragEnd}>
  <SortableContext items={pinnedIds}>
    {pinnedTabs.map(tab => <SortableTab pinned />)}
  </SortableContext>
  <SortableContext items={normalIds}>
    {normalTabs.map(tab => <SortableTab />)}
  </SortableContext>
</DndContext>
```

`handleDragEnd` 只處理同區 reorder → 呼叫 `reorderTabs(newOrder)`。

## 4. Tab 溢出箭頭

### 機制

- 一般區改為 `overflow: hidden`（取代現有 `overflow-x-auto`）
- 左右各一個 chevron 按鈕，有溢出時才顯示
- 箭頭疊在漸層遮罩上方，暗示還有更多 tab

### 顯示邏輯

- 左箭頭：`scrollLeft > 0`
- 右箭頭：`scrollLeft + clientWidth < scrollWidth`
- 用 `ResizeObserver` + `scroll` 事件偵測

### 互動

- 點擊箭頭：平滑捲動一個 tab 寬度
- 滑鼠滾輪 / 觸控板：自然水平捲動
- 捲動後即時更新箭頭顯隱

### 範圍限制

固定區不做溢出處理（固定 tab 數量少且寬度固定，極端情況不處理）。樣式細節到實作時調整。

## 5. 中鍵關閉 + 右鍵選單

### 中鍵關閉

- Tab 上 `onMouseUp`，偵測 `event.button === 1`
- Locked / pinned tab 不回應
- 觸發 `dismissTab`（記錄到 dismissedSessions）

### 右鍵選單

用 `@floating-ui/react` 實作 context menu（已有依賴）。

- Tab 上 `onContextMenu` → 游標位置彈出選單
- 點擊選單外或按 Esc 關閉
- 選單結構可擴充（Phase 2a 加工作區項目）

### 選單項目

| 項目 | 顯示條件 | 動作 |
|------|----------|------|
| 切換至 Terminal | session tab 且 viewMode !== terminal | `setViewMode(id, 'terminal')` |
| 切換至 Stream | session tab 且 viewMode !== stream | `setViewMode(id, 'stream')` |
| — 分隔線 — | | |
| 鎖定分頁 | !locked | `lockTab(id)` |
| 解鎖分頁 | locked && !pinned | `unlockTab(id)` |
| 固定分頁 | !pinned | `pinTab(id)` |
| 取消固定 | pinned | `unpinTab(id)` |
| — 分隔線 — | | |
| 關閉分頁 | 永遠顯示 | locked 時 disabled（灰色） |
| 關閉其他分頁 | 有其他 unlocked tab | dismiss all unlocked except this |
| 關閉右側分頁 | 右側有 unlocked tab | dismiss unlocked tabs to the right |
| — 分隔線 — | | |
| 重新開啟已關閉的分頁 | dismissedSessions.length > 0 | `undismissSession(last)` |

### 行為細節

- Pinned tab 的「解鎖」不顯示（pinned 隱含 locked，要先取消固定）
- 「關閉分頁」在 locked 時以 disabled 狀態顯示，不隱藏（讓使用者知道功能存在但被鎖定）
